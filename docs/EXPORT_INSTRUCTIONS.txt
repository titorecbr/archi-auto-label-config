================================================================================
INSTRUÇÕES PARA EXPORTAR O PLUGIN DO ECLIPSE
================================================================================

Siga estes passos EXATOS para exportar o plugin e deixá-lo pronto para
instalação no Archi:

PASSO 1: IMPORTAR O PROJETO NO ECLIPSE
--------------------------------------
1. Abra o Eclipse IDE
2. File → Import → Existing Projects into Workspace
3. Selecione a pasta do plugin: "/home/victor/Projetos/VH System/Archi Plugin"
4. Marque "Copy projects into workspace" (opcional)
5. Clique em Finish

PASSO 2: CONFIGURAR DEPENDÊNCIAS
---------------------------------
1. Clique com botão direito no projeto → Properties
2. Java Build Path → Libraries → Add External JARs...
3. Navegue até a pasta de plugins do Archi (veja FIND_DEPENDENCIES.md)
4. Selecione estes arquivos:
   - com.archimatetool.model_*.jar
   - com.archimatetool.editor_*.jar
5. Apply and Close

PASSO 3: COMPILAR
------------------
1. Project → Clean → Clean all projects → Build
2. Verifique se não há erros na aba "Problems"
3. Se houver erros, verifique as dependências novamente

PASSO 4: EXPORTAR O PLUGIN
----------------------------
1. File → Export → Plug-in Development → Deployable plug-ins and fragments
2. Selecione o projeto: "com.vhsystem.defaultlabel" (ou "Default Label Plugin")
3. Escolha "Directory" como destino
4. Clique em "Browse" e escolha uma pasta (ex: ~/archi-plugin-export)
5. Marque "Use class files compiled in the workspace"
6. Clique em Finish

PASSO 5: LOCALIZAR O PLUGIN EXPORTADO
---------------------------------------
1. Navegue até a pasta que você escolheu no passo 4
2. Você verá uma estrutura como:
   
   archi-plugin-export/
   └── plugins/
       └── com.vhsystem.defaultlabel_1.0.0.qualifier/
           ├── META-INF/
           ├── plugin.xml
           └── com/...

PASSO 6: INSTALAR NO ARCHI
---------------------------
1. Localize a pasta de plugins do Archi:
   Linux:   ~/.archi/plugins/
   Windows: C:\Users\<usuario>\.archi\plugins\
   macOS:    ~/Library/Application Support/Archi/plugins/

2. Copie a pasta completa:
   com.vhsystem.defaultlabel_1.0.0.qualifier/
   
   Para dentro de ~/.archi/plugins/

3. A estrutura final deve ser:
   ~/.archi/plugins/com.vhsystem.defaultlabel_1.0.0.qualifier/
   ├── META-INF/
   ├── plugin.xml
   └── com/...

PASSO 7: TESTAR
---------------
1. Feche completamente o Archi
2. Abra o Archi novamente
3. Verifique se aparece o menu "Labels Padrão" no menu principal
4. Clique em "Labels Padrão → Gerenciar Labels Padrão"
5. Configure alguns labels padrão
6. Crie um novo elemento sem nome e veja se o label é aplicado

================================================================================
VERIFICAÇÃO FINAL
================================================================================

Após a exportação, verifique se a pasta contém:

✓ META-INF/MANIFEST.MF
✓ plugin.xml
✓ com/vhsystem/defaultlabel/DefaultLabelPlugin.class
✓ com/vhsystem/defaultlabel/LabelManager.class
✓ com/vhsystem/defaultlabel/dialogs/ManageLabelsDialog.class
✓ com/vhsystem/defaultlabel/handlers/ManageLabelsHandler.class

Se todos os arquivos estão presentes, o plugin está pronto para instalação!

================================================================================
PROBLEMAS COMUNS
================================================================================

PROBLEMA: Erro "Cannot resolve com.archimatetool.*"
SOLUÇÃO: Adicione os JARs do Archi ao Build Path (PASSO 2)

PROBLEMA: Plugin não aparece no menu do Archi
SOLUÇÃO: 
- Verifique se copiou a pasta completa para ~/.archi/plugins/
- Verifique Help → Show Log no Archi para erros
- Certifique-se de que reiniciou o Archi completamente

PROBLEMA: Erro ao abrir o diálogo
SOLUÇÃO:
- Verifique se todas as classes foram compiladas
- Verifique se as dependências do Eclipse estão disponíveis
